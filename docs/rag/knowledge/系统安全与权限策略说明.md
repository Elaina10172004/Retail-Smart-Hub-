# 系统安全与权限策略说明

## 文档作用
本文件用于补充阶段二的系统安全知识库，给 AI 提供密码策略、登录限制、会话管理、角色权限和敏感操作校验的统一口径。

## 身份认证与登录限制
1. 当前系统使用账号密码登录，登录成功后签发本地会话 `auth_sessions`。
2. 登录失败限制为连续 `5` 次失败后锁定 `15` 分钟。
3. 被停用的账号不能登录。
4. 登录成功后会重置失败计数和锁定状态。

## 密码策略
1. 密码至少 `8` 位。
2. 大写字母、小写字母、数字、特殊字符四类中至少满足 `3` 类。
3. 新密码不能包含用户名。
4. 管理端新建用户默认密码为 `Retail@123`，首次使用后应尽快修改。

## 密码轮换策略
1. 密码轮换周期为 `90` 天。
2. 距离轮换到期前 `14` 天开始提醒。
3. 仍在使用默认密码的账号会被判定为需要立即轮换。
4. 系统设置中的账号安全区会展示密码龄期、剩余天数和是否应提醒。

## 会话管理
1. 用户可以查看自己的活跃会话。
2. 用户可以移除指定会话，也可以一键移除其他终端会话。
3. 会话移除属于敏感操作，需要输入当前登录密码做二次验证。
4. 当前登录会话不能通过“移除指定会话”接口直接删除。

## 角色与权限
1. 角色和权限采用白名单方式控制。
2. 预置角色不能删除。
3. 仍有关联用户的角色不能删除。
4. 用户必须先停用，才允许删除。
5. 受保护账号不能删除。

## 敏感操作二次验证
以下操作要求输入当前登录密码做二次验证：
1. 移除指定会话
2. 移除其他会话
3. 用户角色调整
4. 角色权限保存
5. 用户删除
6. 角色删除
7. 管理员重置用户密码

## AI 工具权限边界
1. 只读工具仍受后端真实权限控制，不会因为 AI 调用而绕过权限。
2. 订单、库存、采购、财务、客户、审计日志、权限概览、基础资料概览都必须命中相应权限后才会执行。
3. 密码策略、当前密码安全摘要、当前用户会话列表属于账号安全查询，只面向已登录用户本人。
4. 所有 AI 写操作仍然必须走待确认动作链路，不允许直接落库。

## 当前适合进入知识库的问法示例
1. 当前密码策略是什么？
2. 登录失败会锁多久？
3. 帮我看一下我的会话安全状态。
4. 帮我看一下权限与角色概览。
