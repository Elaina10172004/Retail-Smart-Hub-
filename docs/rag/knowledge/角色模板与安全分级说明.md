# 角色模板与安全分级说明

## 文档作用
本文件用于补充阶段二的系统文档知识库，给 AI 提供角色模板、授权分级和安全分级的统一解释口径。

## 角色模板
1. 当前系统采用“角色 + 权限”的白名单控制模式。
2. 预置角色属于系统模板角色，不能直接删除。
3. 自定义角色允许基于模板复制后再调整权限。
4. 角色的 `scope` 用于表示职责范围，不等价于数据隔离。

## 常见角色模板口径
1. 系统管理员
   - 范围：全局
   - 特点：拥有系统设置、权限管理、基础资料和关键业务操作能力
2. 采购相关角色
   - 范围：采购与到货入库
   - 特点：关注采购单、到货验收、入库确认
3. 仓储与发货相关角色
   - 范围：库存与发货
   - 特点：关注库存、出库、物流与消息提醒
4. 财务相关角色
   - 范围：应收应付
   - 特点：关注收款、付款、账龄和经营资金口径
5. 普通业务操作角色
   - 范围：订单或基础查询
   - 特点：通常只有查询或有限创建能力

## 授权模板管理规则
1. 预置角色不能删除。
2. 仍有关联用户的角色不能删除。
3. 权限调整属于敏感操作，需要二次验证。
4. 角色越接近“系统设置、财务、发货、采购执行”，风险越高。

## 安全分级

### 低风险
1. 普通只读查询
2. 仪表盘、报表、库存、客户等查看类操作
3. 不会改变数据库状态

### 中风险
1. 修改个人资料
2. 修改个人密码
3. 移除自己的其他登录会话
4. 会改变账号状态，但不会直接改变核心业务单据

### 高风险
1. 用户角色调整
2. 角色权限保存
3. 用户删除、角色删除、密码重置
4. AI 发起的建单、收款、付款、发货、入库、采购生成等真实写入动作
5. 这些操作必须走权限校验、审计记录，部分还要求二次验证或人工确认

## AI 解释口径
1. 如果问“角色模板”，优先说明当前角色体系和模板约束。
2. 如果问“安全分级”，优先解释低风险、中风险、高风险的边界。
3. 如果问“某操作为什么需要确认”，优先从高风险动作会改动真实业务数据这一点解释。

## 当前适合进入知识库的问法示例
1. 当前系统有哪些角色模板？
2. 角色模板为什么不能随便删？
3. 系统里的高风险操作分级是什么？
4. 为什么 AI 建单和收付款一定要确认？
