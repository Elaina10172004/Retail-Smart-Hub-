# 单据状态流转与操作规则

## 作用
本文件用于补充阶段二知识库，重点说明当前系统里的单据状态、页面快捷操作、消息中心规则和 AI 工具边界。

## 订单规则
1. 销售订单创建后默认进入 `待发货`。
2. 若库存不足，订单会带上 `部分缺货` 的库存状态，但订单仍可进入后续补货流程。
3. 订单只有在 `待发货` 时才允许取消。
4. 订单只有在 `已发货` 时才允许标记完成。
5. 取消订单前会校验是否已有收款记录；若已有收款，不允许直接取消。

## 客户档案规则
1. 客户档案支持新增、修改、查看详情、启用/停用、删除。
2. 删除客户前必须先停用，避免直接误删活跃客户。
3. 客户名称在未删除档案中必须唯一。
4. 订单创建和客户沉淀会持续回写客户累计销售额、累计订单数和最近下单时间。

## 采购规则
1. 采购建议来自低库存和安全库存差额计算。
2. 采购建议仅针对 `active` 状态的商品和供应商生成。
3. 采购单详情包含供应商、来源、金额、到货进度和商品明细。
4. 当前采购页重点支持查看详情、按同供应商筛选、按同状态筛选。

## 消息中心规则
1. 消息中心当前聚合以下真实业务提醒：
   - 库存预警
   - 待发货提醒
   - 采购待处理提醒
   - 应收逾期提醒
   - 最近审计动作
2. 通知会按角色权限过滤，不属于当前角色的提醒不会显示。
3. 已处理通知不会永久保留；当业务状态变化后，通知会在下一次刷新或轮询后自动消失。
4. 前端会定时拉取最新通知，并支持全部标记已读。

## 页面行内操作规则
1. 订单列表的 `...` 菜单应支持：
   - 查看详情
   - 复制建单
   - 按客户筛选
   - 标记完成
   - 取消订单
2. 客户列表的 `...` 菜单应支持：
   - 查看详情
   - 编辑资料
   - 按同渠道筛选
   - 启用/停用
   - 删除客户
3. 采购列表的 `...` 菜单应支持：
   - 查看详情
   - 按同供应商筛选
   - 按同状态筛选

## AI 只读工具补充
当前系统除原有工具外，还应支持更细粒度查询：
1. `get_procurement_detail`
   - 用于查询指定采购单详情
2. `get_customer_detail`
   - 用于查询指定客户详情或客户画像
3. `list_system_notifications`
   - 用于查询当前消息中心待处理提醒

## AI 写操作边界
1. AI 仍不得直接修改角色权限。
2. AI 仍不得跳过人工确认直接执行写操作。
3. 所有受控写操作都必须：
   - 命中权限
   - 参数齐全
   - 生成待确认动作
   - 人工确认后再执行

## 当前推荐问法
1. `帮我查一下采购单 PO-20260311-001 的详情`
2. `帮我看一下客户 CUS-001 的详情`
3. `帮我看一下消息中心有哪些待处理提醒`
4. `朝阳社区店的客户画像是什么`
