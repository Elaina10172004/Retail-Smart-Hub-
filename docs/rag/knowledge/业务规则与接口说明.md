# 业务规则与接口说明

## 文档用途
1. 作为阶段二 RAG 的业务规则补充知识源。
2. 统一说明当前系统中各模块的关键状态流转、权限边界和 AI 工具使用约束。
3. 为后续 Function Use、接口联调和答辩说明提供统一口径。

## 业务定位
1. 当前系统默认面向百货仓储、零售与仓配协同场景。
2. 商品主数据默认以通用 SKU、条码、品类、仓库库存和订单流转为核心，不以汽配型号、车型适配或维修工单为默认前提。
3. 若个别历史文档或示例仍保留早期汽配表述，应以当前百货仓储定位为准。

## 核心业务闭环
1. 销售订单闭环：创建订单 -> 待发货 -> 已发货 -> 已完成。
2. 缺货采购闭环：库存预警 -> 采购建议 -> 生成采购单 -> 到货验收 -> 入库 -> 库存恢复。
3. 财务闭环：订单生成应收 -> 收款登记；采购单形成应付 -> 付款登记。
4. 审计闭环：关键写操作必须写入审计日志，AI 写操作还必须先经过待确认链路。

## 模块状态规则
### 订单
1. 新建订单后自动生成发货单。
2. 订单状态支持：`待发货`、`已发货`、`已完成`、`已取消`。
3. 只有 `待发货` 订单允许取消。
4. 只有 `已发货` 订单允许标记完成。
5. 订单取消前若已有收款记录，则不允许取消。

### 采购与到货
1. 采购建议来源于库存预警与安全库存缺口。
2. 到货单状态常见为：`部分到货`、`已验收待入库`、`已入库`。
3. 推进到货验收后会自动生成或更新待入库单。
4. 采购单全部到货且无待入库单时，采购状态才应更新为 `已完成`。

### 入库
1. 入库确认会同步增加库存。
2. 已入库单不能重复入库。
3. 入库确认属于真实库存变更，必须记审计日志。

### 发货
1. 发货前必须校验库存充足。
2. 发货确认会扣减库存、生成出库记录并写物流信息。
3. 发货单状态支持：`待发货`、`已发货`。
4. 已发货单不能重复发货。

### 财务
1. 应收登记金额不能超过剩余应收金额。
2. 应付登记金额不能超过剩余应付金额。
3. 收付款登记会更新已收/已付金额与最近处理时间。
4. 收付款登记属于高敏感财务动作，AI 必须经过待确认链路。

## 权限口径
1. `orders.view`：查看订单。
2. `orders.create`：创建订单、修改订单状态。
3. `inventory.view`：查看库存、库存调整。
4. `procurement.manage`：采购、到货、入库相关操作。
5. `shipping.dispatch`：发货确认。
6. `finance.receivable`：应收与收款登记。
7. `finance.payable`：应付与付款登记。
8. `reports.view`：经营报表与仪表盘查询。
9. `settings.master-data`：客户、商品、供应商等基础资料维护。
10. `settings.access-control`：用户、角色、权限与审计日志。

## AI 工具口径
### 只读工具
1. 已支持订单、库存、采购、财务、客户、审计日志、报表、仪表盘、到货、入库、发货查询。
2. 只读工具优先根据用户权限决定是否可执行。
3. 只读工具命中后，模型回答应优先引用工具结果，而不是自行猜测实时数据。

### 受控写工具
1. 当前已开放：
   - `create_customer_profile`
   - `generate_shortage_procurement`
   - `advance_arrival_status`
   - `confirm_inbound`
   - `dispatch_shipping`
   - `register_receipt`
   - `register_payment`
   - `create_sales_order`
2. 所有 AI 写操作必须先进入 `ai_pending_actions` 待确认链路。
3. 用户只能确认或取消自己发起的待确认动作。
4. 待确认动作过期后不能执行，需重新发起。
5. 角色权限变更等更高敏感动作当前不开放给 AI。

## AI 提示词建议
1. 查询类尽量提供明确主体：单号、SKU、客户名、供应商名。
2. 写操作尽量使用显式字段：
   - 收款：`登记收款 AR-20260311-001 金额 120`
   - 付款：`登记付款 AP-20260310-001 金额 300`
   - 建单：`创建订单 客户 朝阳社区店 渠道 门店补货 交付日期 2026-03-20 明细 SKU-1001*2, SKU-1002*1`
3. 若参数不完整，系统应优先回问或提示缺少字段，而不是猜测执行。

## 当前限制
1. RAG 仍以本地文档切块和关键词检索为主，尚未接入向量检索。
2. AI 当前还不能直接处理角色权限变更、找回密码等高风险操作。
3. DRL 与需求预测尚未接入当前生产链路。
