# 接口清单与权限映射

## 文档作用
本文件用于补充阶段二的接口知识库，给 AI 提供当前后端主要路由、模块职责和权限映射说明。

## 公共入口
1. 健康检查：`GET /api/health`
2. 登录相关：
   - `POST /api/system/login`
   - `POST /api/system/recover-password`
3. 已登录后会统一进入模块路由。

## 系统模块
1. `GET /api/system/session`
2. `POST /api/system/logout`
3. `POST /api/system/change-password`
4. `POST /api/system/profile`
5. `GET /api/system/notifications`
6. `GET /api/system/sessions`
7. `POST /api/system/sessions/revoke-others`
8. `POST /api/system/sessions/:token/revoke`
9. `GET /api/system/audit-logs`
   - 权限：`settings.access-control`

## 业务模块
1. 仪表盘：`GET /api/dashboard/overview`
   - 权限：`reports.view`
2. 订单：
   - `GET /api/orders`
   - `POST /api/orders`
   - `GET /api/orders/:id`
   - 权限：`orders.view` / `orders.create`
3. 库存：
   - `GET /api/inventory`
   - `GET /api/inventory/overview`
   - `GET /api/inventory/alerts`
   - 权限：`inventory.view`
4. 采购：
   - `GET /api/procurement`
   - `GET /api/procurement/:id`
   - `POST /api/procurement/generate`
   - 权限：`procurement.manage`
5. 到货：
   - `GET /api/arrival`
   - `GET /api/arrival/:id`
   - `POST /api/arrival/:id/advance`
   - 权限：`procurement.manage`
6. 入库：
   - `GET /api/inbound`
   - `GET /api/inbound/:id`
   - `POST /api/inbound/:id/confirm`
   - 权限：`procurement.manage`
7. 发货：
   - `GET /api/shipping`
   - `GET /api/shipping/:id`
   - `POST /api/shipping/:id/dispatch`
   - 权限：`shipping.dispatch`
8. 财务：
   - `GET /api/finance/overview`
   - `GET /api/finance/receivables`
   - `GET /api/finance/receivables/:id`
   - `GET /api/finance/receipts`
   - `POST /api/finance/receivables/:id/receive`
   - `GET /api/finance/payables`
   - `GET /api/finance/payables/:id`
   - `GET /api/finance/payments`
   - `POST /api/finance/payables/:id/pay`
   - 权限：`finance.receivable` / `finance.payable`
9. 报表：
   - `GET /api/reports/summary`
   - `GET /api/reports/overview`
   - 权限：`reports.view`
10. 设置：
   - 用户、角色、权限、供应商、商品、仓库维护相关接口
   - 权限：`settings.access-control` / `settings.master-data`

## AI 模块
1. `GET /api/ai/status`
2. `POST /api/ai/chat`
3. `POST /api/ai/actions/:id/confirm`
4. `POST /api/ai/actions/:id/cancel`

## AI 与权限关系
1. AI 不会绕过真实后端权限。
2. 只读工具命中后仍会再次检查模块权限。
3. 所有写工具都先生成待确认动作，再由用户确认执行。

## 当前适合进入知识库的问法示例
1. 当前财务模块有哪些接口？
2. 到货和入库分别对应哪些路由？
3. 查询审计日志需要什么权限？
4. AI 写操作确认接口是什么？
